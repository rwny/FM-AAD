# แผนการปรับปรุงระบบสู่ BIM-Centric Workflow (v3)

เปลี่ยนจากระบบที่เน้นการป้อนข้อมูลผ่าน Markdown (MD) เป็นการใช้ข้อมูลมาตรฐาน **IFC (Industry Foundation Classes)** ที่สกัดมาจาก Catalog โดยตรงและฝัง (Embed) ลงใน 3D GLB Model

## 1. การเปลี่ยนแปลงหลัก (Key Shifts)
- **Source of Truth:** เปลี่ยนจาก `AR15-AC-DATA.md` เป็น PDF Catalogs ใน `3DModel/catalog2ifc/`
- **Data Carrier:** ข้อมูล Spec ทางเทคนิคจะถูกเก็บไว้ในตัว 3D Model (GLB) ผ่าน `userData` (extras) แทนการอ่านจากไฟล์ JSON แยกเพียงอย่างเดียว
- **Focus:** เริ่มต้นที่ระบบปรับอากาศ (AC System) เป็นลำดับแรก

## 2. ขั้นตอนการดำเนินงาน (Implementation Steps)

### Phase 1: Data Extraction (Extractor)
- สกัดข้อมูลจาก PDF Catalog ของ Carrier (TGF, TGV, TGEV Series)
- แปลงเป็น JSON ฐานข้อมูลกลางที่ `3DModel/catalog2ifc/data/*.js`
- **Output:** ไฟล์ข้อมูล Spec ที่มีค่า BTU, SEER, ขนาด (W/D/H), และรหัสรุ่นที่ถูกต้อง

### Phase 2: BIM Injection (Blender)
- ใช้ **Carrier HVAC Tools (Blender Addon)** ในการวางโมเดลแอร์
- Addon จะทำการ "ฉีด" ข้อมูล IFC เข้าไปใน Mesh:
    - `IfcEntityType`: เช่น `IfcUnitaryEquipment` (FCU), `IfcCondensingUnit` (CDU)
    - `Pset_Carrier.BTU`, `Pset_Carrier.SEER`, `Pset_Carrier.Voltage` ฯลฯ
    - `System_ID`: รหัสเชื่อมโยงระหว่างตัวใน (FCU) และตัวนอก (CDU)
- Export เป็นไฟล์ `ar15-302-bim.glb`

### Phase 3: Web Integration (Vite/React)
- **Model Loader:** ปรับปรุง `BuildingModel.tsx` ให้สามารถอ่าน `mesh.userData` เพื่อดึง Spec ออกมาแสดงผล
- **AC Mode Enhancement:** 
    - เมื่อคลิกที่แอร์ จะแสดงข้อมูลทางเทคนิคเต็มรูปแบบจาก Catalog
    - แสดงการเชื่อมโยง System (คลิกตัวใน ไฮไลท์ตัวนอกที่คู่กัน)
- **Database Sync:** สร้างระบบ Auto-sync เพื่อนำข้อมูลจาก Model ไปลงใน Supabase (Asset Registry) เพื่อเก็บประวัติการซ่อมบำรุง

## 3. ผลกระทบและการเปลี่ยนแปลงโครงสร้าง (Impact Analysis)

| ส่วนประกอบ | สถานะเดิม (v2) | สถานะใหม่ (v3) |
| :--- | :--- | :--- |
| **Data Source** | `AR15-AC-DATA.md` (Manual) | `catalog2ifc` (Automated from PDF) |
| **3D Model** | Simple Meshes (fcu-xxx) | BIM Meshes with IFC Metadata |
| **UI: AC Mode** | แสดงแค่ชื่อรุ่นและสถานะ | แสดง Spec ทางเทคนิคครบถ้วน (BTU, SEER, etc.) |
| **Connectivity** | เช็คจากชื่อไฟล์ | เช็คจาก `System_ID` ใน Metadata |
| **Workflow** | MD -> JSON -> App | PDF -> JSON -> Blender -> GLB -> App |

## 4. แผนการพัฒนา AC Mode (V3 Priority)
1. **Model Update:** อัปเดต GLB ให้มีข้อมูล `userData` ตามมาตรฐาน IFC
2. **Spec Viewer:** พัฒนา UI Sidebar ให้ดึงค่าจาก `userData` มาแสดง
3. **System Linking:** ฟีเจอร์ "Show Connected Unit" เพื่อดูทิศทางการเชื่อมต่อระหว่าง FCU และ CDU
