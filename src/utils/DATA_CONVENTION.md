# BIM Data Convention (AR15.md)
คู่มือการบันทึกข้อมูลโครงสร้างอาคารและสินทรัพย์ เพื่อให้สคริปต์แปลงข้อมูลเข้าสู่ระบบ 3D ได้ถูกต้อง

---

## 1. โครงสร้างลำดับชั้น (Hierarchy)
ใช้การย่อหน้า (Indentation) 4 spaces หรือ 1 Tab ตามมาตรฐาน Markdown:
- **Floor** (ระดับ 1) : ต้องมีคำว่า `FLOOR`
    - **Room** (ระดับ 2) : ต้องขึ้นต้นด้วย `rm-` (เช่น `rm-101`)
        - **System** (ระดับ 3) : `AR`, `FUR`, `EE`, `AC`
            - **Category** (ระดับ 4) : `LF` (Loose Furniture), `BF` (Built-in)
                - **Type Group** : ควรมีหมวด `Type` ไว้ด้านบนเพื่อนิยามรุ่นหลัก

---

## 2. การตั้งชื่อรหัส (Main Type vs Instance)
ระบบแยกข้อมูลออกเป็น 2 ส่วน เพื่อลดการเขียนข้อมูลซ้ำซ้อน:

### 2.1 รุ่นหลัก (Main Type) : ใช้ก้ามปู `[...]`
ใช้เพื่อนิยาม **"สเปคกลาง"** ของเฟอร์นิเจอร์รุ่นนั้นๆ
- **รูปแบบ:** `[LF-X]` (เช่น `[LF-1]`, `[LF-4]`)
- **ตำแหน่ง:** ควรเขียนไว้ภายใต้หมวด `Type` ของแต่ละ Category
- **ข้อมูลภายใน:** ใส่ยี่ห้อ (Brand), รุ่น (Model), หรือประวัติการเปลี่ยนสเปครายปี
- *Instance ทุกตัวที่ใช้รหัสขึ้นต้นเหมือนกัน จะดึงข้อมูลจากตรงนี้ไปใช้เป็นค่าเริ่มต้นอัตโนมัติ*

### 2.2 ชิ้นงานจริง (Instance) : ใช้จุดทศนิยม `.n`
ใช้แทน **"วัตถุแต่ละชิ้น"** ที่วางอยู่ในห้องจริง
- **รูปแบบ:** `LF-X.N` (เช่น `LF-1.1`, `LF-1.2`)
- **ตำแหน่ง:** เขียนไว้ภายใต้ห้อง (Room) ที่วัตถุนั้นตั้งอยู่จริง
- **ข้อมูลภายใน:** ใส่ข้อมูลเฉพาะตัว เช่น `Asset-ID` (รหัสพัสดุ), วันที่ติดตั้งจริง หรือบันทึกการซ่อม (Status-log) ของชิ้นนั้นๆ

---

## 3. การจัดการวงจรชีวิตวัตถุ (Object Lifecycle)
สคริปต์ใช้หลักการ **"Latest Date Wins"** (วันที่ล่าสุดคือตัวปัจจุบัน):
- คุณสามารถเขียนรหัสเดียวกัน (เช่น `LF-1.1`) ซ้ำหลายที่ได้ (เช่น แยกตามปีงบประมาณ)
- สคริปต์จะคำนวณวันที่จาก:
    1. ฟิลด์ `Install : YYYY-MM-DD`
    2. วันที่ล่าสุดใน `Status-log`
- **ตัวที่วันที่ใหม่ที่สุด:** จะถูกนับเป็น "Active Asset" และแสดงข้อมูล Spec หลัก
- **ตัวที่วันที่เก่ากว่า:** จะถูกเก็บเข้าเป็น "History" (ประวัติ) โดยอัตโนมัติ

---

## 4. สถานะและสี (Status & Keywords)
ระบบจะตรวจจับคำสำคัญในฟิลด์ `status` หรือใน `Status-log` เพื่อเปลี่ยนสีใน 3D และสรุปยอด:

| สถานะ | ตัวอย่าง Keyword | สีใน 3D | สีใน Log |
| :--- | :--- | :--- | :--- |
| **ปกติ** | `Normal`, `install`, `ปกติ` | ฟ้า/น้ำเงิน | 🟢 เขียว |
| **รอซ่อม** | `maintenance`, `warning`, `ซ่อม`, `บำรุง` | ส้ม | 🟡 ส้ม |
| **ชำรุด/เลิกใช้** | `faulty`, `broken`, `พัง`, `หัก`, `เลิกใช้` | แดง/เทา | 🔴 แดง |

> **หมายเหตุ:** หากระบุสถานะว่า **"เลิกใช้"** ระบบจะ **ไม่นับจำนวน** รวมในห้องนั้น แต่ยังคงแสดงรายชื่อเป็นตัวจางๆ ในประวัติ

---

## 5. ฟิลด์ข้อมูลมาตรฐาน (Metadata)
เพื่อให้การแสดงผลสวยงาม ควรใช้คีย์ดังนี้ (Case-Sensitive):
- `Asset-ID :` รหัสพัสดุ (สถ.xxxx)
- `Brand :` ยี่ห้อ
- `Model :` รุ่น
- `Install :` วันที่ติดตั้ง (รูปแบบ YYYY-MM-DD)
- `Status-log` : หัวข้อสำหรับบันทึกประวัติ (ย่อหน้าลงไปอีกชั้น)

---

## 6. วิธีการอัปเดตข้อมูล
1. แก้ไขข้อมูลในไฟล์ `src/utils/AR15.md`
2. บันทึกไฟล์ (Save)
3. รันคำสั่งแปลงข้อมูลใน Terminal:
   ```bash
   node scripts/build-ar15-md-to-json.js
   ```
4. รีเฟรชหน้าเว็บเพื่อดูผลลัพธ์
