# หลักการบันทึกข้อมูลอาคาร (BIM/FM Data Convention)
คู่มือการบันทึกข้อมูลด้วยรูปแบบ Markdown เพื่อแปลงเป็น JSON สำหรับใช้งานในระบบ 3D/AR

## 1. โครงสร้างลำดับชั้น (Indentation Rule)
การเว้นวรรค (Indent) คือหัวใจสำคัญในการระบุว่าข้อมูลนั้นสังกัดอยู่ภายใต้หัวข้อใด **ต้องใช้ 4 spaces ต่อ 1 ระดับ**

- LEVEL 0: ชื่ออาคาร (เช่น `- AR15`)
    - LEVEL 1: ชั้น (เช่น `- FLOOR 1`)
        - LEVEL 2: ห้อง (เช่น `- rm-101 : ชื่อห้อง`)
            - LEVEL 3: ระบบงาน (เช่น `- FUR`, `- EE`, `- AC`)
                - LEVEL 4: หมวดหมู่ย่อย (เช่น `- LF` ย่อมาจาก Loose Furniture)
                    - LEVEL 5: นิยามประเภททรัพย์สิน (`- [TYPE-ID]`) หรือ ตัวทรัพย์สินจริง (`- INSTANCE-ID`)

---

## 2. การนิยามประเภท (Asset Type Definition)
ใช้สำหรับเก็บข้อมูลส่วนกลางที่ทรัพย์สินรุ่นเดียวกันมีเหมือนกัน เพื่อไม่ต้องเขียนซ้ำ
- **รูปแบบ:** `- [ID-NAME]` (มีวงเล็บก้ามปู)
- **ตัวอย่าง:**
    ```markdown
    - [LF-1]
        - Type : โต๊ะ 4 ขา
        - Brand : ModernForm
        - Model : MD-1565465
    ```

---

## 3. การบันทึกทรัพย์สินรายชิ้น (Asset Instance)
คือของที่มีอยู่จริงในห้องนั้นๆ โดยอ้างอิง ID มาจากประเภทข้างต้น
- **รูปแบบ:** `- ID.ลำดับ` (เช่น `LF-1.1`, `LF-1.2`)
- **ข้อมูลภายใน:**
    - `Asset-ID` : รหัสทะเบียนพัสดุ/ครุภัณฑ์
    - `Install` : วันที่ติดตั้ง
    - `Status-log` : ประวัติการใช้งาน/ซ่อมบำรุง

---

## 4. การบันทึกประวัติ (Status Log)
บันทึกเหตุการณ์ที่เกิดขึ้นตามลำดับเวลา
- **รูปแบบ:** `- YYYY-MM-DD : อาการ/เหตุการณ์`
- **บันทึกเพิ่มเติม:** หากมีรายละเอียดมาก ให้เว้นวรรคเข้าไปอีก 1 ระดับใต้บรรทัดวันนั้นๆ
- **ตัวอย่าง:**
    ```markdown
    - Status-log
        - 2026-03-12 : ซ่อมขา
            - ดำเนินการโดยช่างวิชัย ใช้กาวตราช้าง
    ```

---

## 5. ข้อควรระวัง (Do's & Don'ts)
- ✅ **Do:** ใช้เครื่องหมาย `:` คั่นระหว่าง Key และ Value โดยมีช่องว่างเสมอ (เช่น `Key : Value`)
- ✅ **Do:** รักษาระดับ Indent ให้ตรงกันตลอดทั้งไฟล์
- ❌ **Don't:** ห้ามใช้ Tab (ให้ใช้ Space 4 ครั้งแทน) เพื่อป้องกันความเพี้ยนในบาง Editor
- ❌ **Don't:** หลีกเลี่ยงการใช้เครื่องหมายพิเศษอื่นๆ ในชื่อ ID นอกจาก `-` และ `.`

---

## ตัวอย่างโครงสร้างที่สมบูรณ์
```markdown
- AR15
    - FLOOR 1
        - rm-101 : ห้องประชุม
            - FUR
                - LF
                    - [LF-1]
                        - Type : โต๊ะทำงาน
                    - LF-1.1
                        - Asset-ID : 68-001
                        - Status-log
                            - 2026-01-01 : ติดตั้งใหม่
```
