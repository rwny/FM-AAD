# 🚀 Future Expansion Roadmap (BIM-FM Digital Twin)

*ร่างรายการแผนการพัฒนาที่ต่อยอดจาก `MASTER_SYSTEM_ARCHITECTURE.md` ฉบับปัจจุบัน เพื่อรองรับ Scale-up ในอนาคต*

---

## 🏗️ 1. Multi-Building & Faculty Scale-Up (ขยายระดับองค์กร)
- [ ] **ปรับโครงสร้าง `AR15-DATA.md`:** ยกระดับให้รองรับ Root Node ระดับกลุ่มอาคาร เช่น `- Faculty of Architecture` และซ้อน `<Building-ID>` เข้าไปด้านล่าง
- [ ] **อัปเดต Parser Script:** แก้ไข `sync-ac-to-supabase.cjs` ให้รู้จักเงื่อนไขและแยกแยะประเภท Node `type: 'faculty'` และเพิ่มเงื่อนไขตรวจจับชื่ออาคาร (เช่น ขึ้นต้นด้วย `AR-`) เป็น `type: 'building'`
- [ ] **3D Context Switcher:** สร้าง UI (Dropdown/Sidebar) บนแดชบอร์ดเพื่อให้ผู้ใช้งานเลือกโหลด Scene 3D `.glb` แยกตามอาคาร เพื่อรักษาประสิทธิภาพ (Performance) การเรนเดอร์ภาพ

## 🧩 2. System-Centric Expansion (ขยายระบบย่อยในอาคาร)
- [ ] **Data Hierarchy Integration:** กำหนดระดับ "คลัสเตอร์ระบบ" ลงในแต่ละห้อง (Room) หรือชั้น (Floor) โดยแบ่งเป็น 5 ระบบมาตรฐาน:
  - `ARCH` (สถาปัตยกรรม)
  - `FUR` (ครุภัณฑ์เฟอร์นิเจอร์)
  - `EE` (ระบบไฟฟ้า)
  - `AC` (ระบบปรับอากาศ - *ใช้งานอยู่*)
  - `CCTV` (ระบบกล้องวงจรปิด)
- [ ] **Frontend Integration:** เพิ่มปุ่มเมนูสำหรับสลับโหมด (AR, Fur, CCTV) เข้าไปในแถบเมนูหลักของหน้าต่าง 3D
- [ ] **CCTV Strategic Placement:** ควบคุมการโยงเส้นระบบ CCTV พื้นที่ส่วนกลาง ให้เกาะอยู่กับระดับ **- ar15** (สำหรับ NVR) หรือ **- floor** (สำหรับกล้องโถงทางเดิน) โดยไม่ต้องนำไปผูกติดกับกำแพงห้องใดห้องหนึ่ง

## 📎 3. Product Catalog & Manual Integration
- [ ] **Global Dictionary Link:** นำลิงก์ PDF แคตตาล็อกกลาง ใส่ไว้ในไฟล์โครงสร้างสเปค (เช่น `src/data/carrier-tgf.ts` -> `pdfUrl: "Carrier-40TGF.pdf"`) เพื่อลดความซ้ำซ้อน
- [ ] **Instance-Specific Documents:** สำหรับใบรับประกันหรือเอกสารเฉพาะเจาะจงรายเครื่อง ให้เพิ่มคีย์ `CatalogLink: "url-to-pdf"` แทรกในปีกกา `{ }` ของไฟล์ `.md` เพื่อให้สคริปต์โยนเข้า Metadata บนฐานข้อมูล

## 🛠️ 4. Maintenance & Service Logs Upgrade
- [ ] **Contractor/Technician Tracking:** เมื่อมีการจ้างบริษัทซ่อมบำรุงภายนอก (Outsource) ให้เพิ่มตารางหรือคอลัมน์ใหม่ใน Supabase (เช่น `contractor` หรือ `technician_name`) ในฐานข้อมูล `ac_logs`
- [ ] **UI Update for Logs:** นำคอลัมน์ใหม่มาเชื่อมต่อเพื่อแสดงผล "ชื่อบริษัท/ช่าง" บนหน้าต่าง Timeline ขวามือเมื่อผู้ใช้เรียกดูประวัติการซ่อมบำรุงของอุปกรณ์ชิ้นนั้นๆ

## 🌐 5. True-BIM (IFCOpenShell) Integration [วิสัยทัศน์ระยะยาว]
- [ ] **IFC GUID Mapping:** ในกรณีที่มีการใช้ไฟล์สถาปัตยกรรมต้นฉบับที่เป็นมาตรฐานเปิด `.ifc` จากผู้จัดทำโมเดล แทนที่จะตั้งรหัสชิ้นส่วนเอง (เช่น `AC-101-1`) ให้นำรหัสสากล (GlobalId แบบ 22 ตัวอักษร) มาฝังเป็น `IFC_GUID: 1wB_...` ลงในไฟล์ข้อมูล
- [ ] **Automated Sync Engine:** หากระบบก้าวหน้าถึงขีดสุด อาจมีการเขียนสคริปต์สกัดรหัส IFC_GUID จากโมเดลสถาปัตยกรรมได้โดยตรง เพื่อขจัด Human Error ในการตั้งชื่อไฟล์ 100%

## ♻️ 6. Asset Lifecycle & Replacement (วงจรชีวิตและการสับเปลี่ยนครุภัณฑ์)
- [ ] **Slot-Based Architecture:** สร้างกฎระเบียบ (Data Convention) ให้การแก้ไขไฟล์ `.md` แยก "จุดติดตั้ง" (เช่น `AC-101-1`) ออกจาก "ตัวเครื่องจริง" (ผ่าน `AssetID`) เพื่อรับประกันว่า 3D Model จะไม่แตกพังเมื่อมีการเปลี่ยนแผงงวงจรหรือซื้อแอร์ตัวใหม่
- [ ] **Decommissioned Zone per Building:** ทุกครั้งที่อุปกรณ์ถูกโละทิ้ง ให้ย้ายข้อมูลไปวางไว้ในกลุ่ม `- DECOMMISSIONED_<ชื่ออาคาร>` โดยจัดให้อยู่ภายในโครงสร้างของอาคารนั้นๆ (เช่น ให้อยู่ใต้ `- ar15` หรือ `- ar1` แทนที่จะเอาไปกองรวมกัน) เพื่อง่ายต่อการรักษากราฟความสัมพันธ์ว่าซากอุปกรณ์นี้เคยประจำการอยู่ที่อาคารไหน

## 📅 7. Smart Space Management & Scheduling (บริหารพื้นที่และเวลาใช้งาน)
- [ ] **Room Schedule Integration:** บันทึกข้อมูลตารางเรียนหรือสถานะการจองลงในระดับ `room` (เช่นเพิ่มคีย์ `Status: "Occupied"` หรือลิงก์ `ScheduleLink` ในวงเล็บปีกกา)
- [ ] **Real-time Occupancy Heatmap:** นำข้อมูลตารางสอนไปย้อมสี 3D Model ตามเวลาจริง (เช่น สีเขียว = ห้องว่าง, สีส้ม = มีเรียนอยู่) เพื่อดู Heatmap ความหนาแน่นของทั้งอาคารด้วยตาเปล่า
- [ ] **Energy Waste Detection:** สร้าง Logic วิเคราะห์ข้ามระบบ เช่น ถ้าตารางเรียนระบุว่าห้อง "ว่าง" แต่ข้อมูลในระบบปรับอากาศดันฟ้องว่า "แอร์เปิดอยู่" ให้เด้งการแจ้งเตือนสีแดง (Alert) ขึ้นมาเตือนให้ รปภ. เดินไปปิดแอร์เพื่อประหยัดไฟ

## 👁️ 8. Advanced Visualization & Spatial Knowledge Graph (การผสานกราฟกับพื้นที่ 3D)
- [ ] **Bi-directional Selection Sync:** พัฒนา Shared Global State (Zustand/Context) เพื่อเชื่อมโยงการเลือก:
  - เมื่อคลิกโหนดใน **Graph** -> กล้อง 3D จะบิน (Tween) ไป Focus และ Highlight ที่ **Mesh** ในอาคารทันที
  - เมื่อคลิกที่อุปกรณ์ใน **3D Model** -> กราฟจะทำการขยาย (Expand) และ Focus โหนดที่เกี่ยวข้องเพื่อโชว์ Metadata
- [ ] **Real-time Status Sync (Mesh Color):** พัฒนาระบบย้อมสีโมเดล 3D ตามสถานะใน Database:
  - หากอุปกรณ์ในกราฟ "กระพริบแดง (Faulty)" -> ตัวเครื่องในโมเดล 3D (Blender Mesh) จะต้อง "กระพริบแดง" ตามไปด้วย เพื่อให้ช่างเห็นตำแหน่งทางกายภาพที่เสียได้ทันที
- [ ] **Spatial Edge Visualization (Digital Pipe):** นำเส้นความสัมพันธ์ (Edges) ใน Knowledge Graph มาวาดเป็นเส้นเรืองแสง (Shader Path) ใน 3D Model:
  - แสดงทิศทางการเชื่อมต่อของท่อน้ำยา/ท่อลม (Refrigerant/Air Flow) ที่วิ่งทะลุผนังจริงๆ ตามความสัมพันธ์ `connectsTo` หรือ `monitors`
- [ ] **X-Ray Maintenance Mode:** สร้างปุ่มสลับโหมดที่ทำให้ผนังอาคารโปร่งใส (Transparent) เพื่อให้เห็นระบบงานระบบ (MEP) ที่ซ่อนอยู่หลังฝ้า/ผนัง โดยอ้างอิงตำแหน่งจากกราฟความสัมพันธ์
